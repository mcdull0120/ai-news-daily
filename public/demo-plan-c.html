Claude Code v2.1.142：MCP 增强 + Fast 模式升级
点击展开
2026.05.14·GitHub Releases

Claude Code 最新版本新增了 8 个命令行配置标志，包括 --add-dir、--settings、--model 等，方便配置后台会话。Fast 模式的默认模型升级为 Opus 4.7。插件功能也增强了——拥有根目录 SKILL.md 的插件现可被识别为技能。

🔍 背景补充

MCP 是什么？Model Context Protocol（模型上下文协议），是 Claude 的"插件接口"。通过 MCP，Claude Code 可以连接外部工具（比如查新闻、操作数据库、发邮件）。这次更新让 MCP 工具的超时设置更稳定，意味着复杂操作不容易中途断了。

SKILL.md 被识别为技能意味着什么？开发者以后给 Claude Code 写扩展更容易了——只要在项目根目录放一个 SKILL.md 文件，Claude Code 就能自动识别并加载其中的功能。门槛大幅降低。

趣味💬 X：宝玉
为什么资深开发者讲不清自己的专业能力？
点击展开
2026.05.15·X：宝玉 (@dotey)

一篇很有意思的长文。作者说，业务团队活在"消除不确定性"的循环里——要的是快、是验证。而资深开发者活在"管理复杂性"的循环里——要的是稳、是不崩。两边语言不通，所以开发者说"这个需求会增加系统复杂度"，业务听成了"他不想干"。文章最后指出：AI 能写代码，但没法"承担责任"。

🔍 背景补充

作者是谁？宝玉（@dotey），前阿里技术专家，现在在 AI 领域做独立观察。他的文章特点是：不讲技术细节，讲"技术人怎么看世界"。

为什么这个话题突然火了？因为 AI 编程工具（Claude Code、Cursor）越来越强了，很多人开始问："如果 AI 能写代码，还要开发者干什么？"这篇文章的回答是：开发者的核心价值不是"写代码"，而是"为系统长期稳定承担责任"。

为什么值得关注？

  • AI 能写代码，但没法"承担责任"。资深开发者不可替代的价值在于：系统塌了的时候，有人能兜底。AI 没有背锅能力，这是它和人之间最本质的差距。
  • 对小白用户的启示。你不需要成为"资深开发者"才能用 Claude Code。你要学的是怎么把任务拆清楚、怎么验收结果——这些能力，业务思维和工程思维都需要。
