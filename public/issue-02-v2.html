🔥 今日事件

Anthropic 首次在 B2B 采用率上超越 OpenAI

来源：Ramp AI 支出数据 · 阅读原文 → （英文，需翻墙）

这是什么？

美国企业 AI 支出数据出炉了：用 Claude（Anthropic）的企业占 34.4%，首次超过用 ChatGPT（OpenAI）的 32.3%。而且 Claude 的企业客户数量在一年内翻了 4 倍。

为什么值得关注？

  • 企业买单的逻辑变了。以前谁名气大谁赢，现在谁能真正"干活"谁赢。Claude 在编程、文档处理、复杂推理上的 Harness（驾驭能力）更强，所以企业愿意付费。
  • B2B 市场看性价比和可控性。Claude 的 API 价格策略更激进，对企业数据安全、合规性的支持更好。
  • 但领先优势可能不稳。OpenAI 正在追赶，Google Gemini 在 B2B 也在发力，企业采购决策有滞后性。

可以学到什么？

这条数据证明了一件事："工具好不好用"（Harness）正在比"模型名不名气"（品牌）更重要。 Claude 能反超，不是因为 Claude 模型比 GPT-4 聪明很多，而是因为 Claude 的"身体"（Claude Code、API 稳定性、企业功能）更适合干活。

延伸启发

对个人用户来说，选 AI 工具时不要只看"这个模型厉不厉害"，要看"这个工具能不能帮我真正解决问题"。一个智商 120 但会动手的助理，比智商 140 但只会动嘴的顾问更有价值。

📚 概念学习

Harness（驾驭能力）到底是什么？

上一期我们讲了"大脑"和"身体"的区别。这期讲一个更深层的问题：为什么"身体"的好坏，往往比"大脑"更能决定你的体验？

一、从"聊天"到"干活"的进化

想象两个助理：

助理 A（普通对话 AI）：智商 140，哈佛毕业。你让他写个 PPT，他说："好的，这是 PPT 的结构大纲和内容。"然后... 没了。你自己打开 PowerPoint，复制粘贴，调格式，找配图。

助理 B（Claude Code）：智商 120，普通大学毕业。你让他写个 PPT，他说："好的。"然后他自己打开 Gamma/Canva，生成 PPT，调好格式，插好图，发你链接。如果你说"字体太丑"，他自己改，改完再发你。

你选谁？大多数人选 B。因为 B 的 Harness（驾驭系统）好——他不只是"会"，他是"能干"。

二、Harness 的 5 个核心组件

Harness 不是单一功能，而是一套"让大脑能干活"的系统：

1
环境感知——读取周围的信息和文件。Claude Code 能自动扫描你整个代码库，理解项目结构；普通对话 AI 只能你手动复制粘贴代码给他。
2
文件操作——读写、创建、删除文件。Claude Code 直接创建 index.html、style.css；普通对话 AI 给你代码文本，你自己保存。
3
终端执行——运行命令、测试、部署。Claude Code 能运行 npm install、启动服务器、跑测试；普通对话 AI 无法执行任何命令。
4
多步规划——把大任务拆成小步骤。"做网站"→建文件→写代码→调样式→测试→修复。普通对话 AI 只能单轮回答，没有长期记忆和规划。
5
工具调用——调用外部 API、数据库、服务。Claude Code 通过 MCP/Skill 查新闻、操作数据库、发邮件。这就像 iPhone 的 App Store，装了 App 就无所不能。

关键洞察：组件 1-3 是"基础体力"（让 AI 从动嘴变动手）；组件 4 是"脑力升级"（从回答问题变完成任务）；组件 5 是"无限扩展"（从单机变联网）。

三、为什么 Claude Code 的 Harness 特别强

同样是"身体"，Claude Code 的身体为什么比别的工具强？三个原因：

1. 自主执行循环（Agentic Loop）

普通工具的流程：你说 → 它答 → 结束。

Claude Code 的流程：你说 → 它理解 → 拆步骤 → 执行第 1 步 → 看结果 → 发现错误 → 自我修正 → 执行第 2 步 → ... → 完成 → 汇报。

这就像：普通 AI 是"你问一句，我答一句"的客服；Claude Code 是"你给我目标，我自己搞定"的项目经理。

2. 上下文窗口管理

大模型都有"上下文窗口"（一次能看多少字的限制）。Claude Code 的 Harness 会自动判断：这个任务需要看哪些文件？文件太长哪部分最重要？多轮对话后哪些信息不能丢？普通工具需要你手动管理这些（复制粘贴、反复提醒），Claude Code 自己搞定。

3. 工具生态（MCP + Skill）

Claude Code 不靠自己学会所有技能，而是通过标准化接口接入无限的外部工具：需要查新闻？装 AI HOT Skill。需要操作数据库？装数据库 MCP。这就像给厨师长配了一个"甜点师"助手，他原来不会做蛋糕，现在能做了。

四、国内模型 vs 国外模型的真实差距

上一期你说"我用的模型还是国内的模型"。这期回答你：差距到底在哪？值不值得"换发动机"？

直观感受的差距：

维度 国外顶尖 国内主流
回答质量 逻辑严密、举例贴切 日常 OK，复杂问题易"胡说"
创造力 写故事、想点子很强 偏保守，容易"标准答案化"
代码能力 能写复杂项目、调试、优化 能写简单代码，复杂项目易出错
长文本 能一次处理几十万字 通常 4K-16K，长文档易"遗忘"
诚实度 不知道就承认 有时会"幻觉"（自信地胡说）

注意：这个对比是"顶尖国外模型" vs "国内平均水平"。国内也有很强的模型（DeepSeek、Kimi、通义千问最新版），差距在缩小。

技术差距的根源：

  • 训练数据：国外模型覆盖全球互联网，代码、论文、多语言内容极其丰富；国内模型中文数据强，但英文技术文档、开源代码覆盖相对弱。
  • 对齐技术：国外模型在"指令跟随"和"安全对齐"上投入更多；国内模型有时"对齐过度"——为了安全拒绝回答正常问题，或回答很模板化。
  • 推理能力：国外模型在复杂数学、逻辑推理、多步规划上更稳定；国内模型"一步推理"还行，"多步推理"容易在中间某步出错。

给你的实用建议：

国内模型够用：日常聊天、查资料、写简单文案、处理中文内容（古文、诗词、国内政策）、对数据隐私要求极高。

建议用国外模型：编程开发、复杂推理/数据分析、长文档处理、创意工作、用 Claude Code / Cursor 等工具时。

性价比方案：日常用国内（快+便宜），重要工作用国外（准+强），编程用 Claude Code。

差距不在"懂不懂"，在"能不能干"。 Harness 决定了一个 AI 是"顾问"还是"员工"。

💡 灵魂洞察

智商高的顾问很多，但能把事办成的员工稀缺。

选 AI 工具，先看它能不能干活，再看它聪不聪明。
