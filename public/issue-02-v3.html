02

📚 概念 & 案例

先搞懂原理，再看真实对比

上一期我们聊了"大脑"和"身体"的区别。但说实话，很多朋友看完还是懵——为什么同样的"大脑"（比如 GPT-4），装在 Claude Code 这个"身体"里体验这么好，放在别的地方就一般？

我跟你说，核心差异就在一个东西上：Harness（驾驭能力）。它为什么比"智商"更重要？今天聊清楚。

一、从"聊天"到"干活"的进化

想象两个助理：

助理 A（普通对话 AI）：智商 140，哈佛毕业。你让他写个 PPT，他说："好的，这是大纲和内容。"然后... 没了。你自己打开 PowerPoint，复制粘贴，调格式，找配图。累不累？

助理 B（Claude Code）：智商 120，普通大学毕业。你让他写个 PPT，他说："好的。"然后自己打开 Gamma/Canva，生成 PPT，调好格式，插好图，发你链接。你说"字体太丑"，他自己改，改完再发你。

你选谁？反正我选 B。因为 B 的 Harness（驾驭系统）好——他不只是"会"，他是"能干"。

二、Harness 的 5 个核心组件

Harness 不是单一功能，它其实是一套"让大脑能干活"的系统。我把它拆成 5 个部分，你会发现生活里处处都是这套逻辑：

1
环境感知——读取周围的信息和文件。Claude Code 能自动扫描你整个代码库，理解项目结构；普通对话 AI 只能你手动复制粘贴代码给他。
2
文件操作——读写、创建、删除文件。Claude Code 直接创建 index.html、style.css；普通对话 AI 给你代码文本，你自己保存。
3
终端执行——运行命令、测试、部署。Claude Code 能运行 npm install、启动服务器、跑测试；普通对话 AI 无法执行任何命令。
4
多步规划——把大任务拆成小步骤。"做网站"→建文件→写代码→调样式→测试→修复。普通对话 AI 只能单轮回答，没有长期记忆和规划。
5
工具调用——调用外部 API、数据库、服务。Claude Code 通过 MCP/Skill 查新闻、操作数据库、发邮件。这就像 iPhone 的 App Store，装了 App 就无所不能。

一句话总结：组件 1-3 是"基础体力"（让 AI 从动嘴变动手）；组件 4 是"脑力升级"（从回答问题变完成任务）；组件 5 是"无限扩展"（从单机变联网）。

三、为什么 Claude Code 的 Harness 特别强

同样是"身体"，Claude Code 的身体为什么比别的工具强？我跟你说，就三个原因：

1. 自主执行循环（Agentic Loop）

普通工具的流程：你说 → 它答 → 结束。

Claude Code 的流程：你说 → 它理解 → 拆步骤 → 执行第 1 步 → 看结果 → 发现错误 → 自我修正 → 执行第 2 步 → ... → 完成 → 汇报。

这种感觉就像：普通 AI 是"你问一句，我答一句"的客服；Claude Code 是"你给我目标，我自己搞定"的项目经理。你品品这个差别。

2. 上下文窗口管理

大模型都有个限制，叫"上下文窗口"——一次能看多少字。Claude Code 的 Harness 会自动判断：这个任务要看哪些文件？文件太长哪部分最重要？多轮对话后哪些信息不能丢？普通工具你得手动管这些，复制粘贴、反复提醒，累得要死。Claude Code 自己搞定。

3. 工具生态（MCP + Skill）

Claude Code 不靠自己学会所有技能，而是通过标准化接口接入外部工具：需要查新闻？装 AI HOT Skill。需要操作数据库？装数据库 MCP。这就像给厨师长配了个"甜点师"助手，他原来不会做蛋糕，现在能做了。

光说不练假把式。上面讲的都是理论，现在我们来看一个真实对比——同一个需求，三种工具分别怎么回应。

实验场景

"帮我分析一下这份销售数据，找出问题并给出建议"

国内模型（网页版）

[给你一段通用的销售分析框架]

建议：加强客户关系管理、优化产品线...

❌

没有读你的数据文件。建议是"套话"，不针对你的具体情况。它"懂"销售分析，但"干不了"分析你这份数据的事。

GPT-4（网页版）

[你手动复制粘贴数据后]

能指出几个数字异常。

⚠️

模型智商够，能分析。但它"身体"受限：数据量大时复制粘贴很麻烦；想生成图表只能给你代码，你自己运行。

Claude Code + Claude 模型

→ 读取 sales_2025.csv ... 完成（自动扫描文件）

→ 发现数据量 50 万行，自动采样分析 ... 完成

→ 发现 3 个异常：Q3 华南区下滑 40%、新品 A 退货率 25%、老客户复购率下降

→ 生成可视化图表 ... 完成

→ 给出针对性建议：华南区建议调研竞品 X、新品 A 建议暂停推广

→ 生成完整分析报告 report.html ... 完成

→ 打开浏览器展示报告

✅

自动读文件、处理大数据、生成图表、输出报告。不是"模型懂不懂分析"的问题，而是"工具能不能干"的问题。

差距不在"懂不懂"，在"能不能干"。

Harness 决定了一个 AI 是"顾问"还是"员工"。

本节小结

Harness 的本质就是一套"执行力系统"。它不改变模型的"智商"，但决定了这个智商能不能真正落地。

03

💡 灵魂洞察

不只关于技术，关于人与工具

我们总在追求更聪明的 AI，

却忽略了更重要的问题：

"它愿不愿意、能不能帮你把事办完？"

两个木匠比赛做椅子。一个懂力学原理、人体工学，但只动嘴不拿锯。另一个理论一般，但拿起木头就锯、钉、打磨，错了就改。最后，第二把椅子摆在面前。

你坐哪一把？

今天的 AI 行业正经历同样的分化：OpenAI 们在拼"智商"（参数、考试分数），Anthropic 在拼"执行力"（能不能把活干完）。

说真的，很多人用 AI 时有个隐秘的焦虑：用多了，我会不会变笨？

我自己的感受是，关键不在"用不用"，而在你怎么用。把 AI 当"代笔者"，你确实会退化——从"做"变成了"审"，从"创造"变成了"验收"。但如果你把 AI 当"杠杆"，你做判断和创意，它做执行，你的能力会放大十倍。

Harness 的本质就是杠杆。它不改变你的智商，但把你的动手能力放大百倍。当然，杠杆也能放大错误。方向错了，它会更快、更完美地帮你走错路。

所以真正重要的，不是"会不会用 AI"，而是"知不知道要做什么"和"知道对不对"。模型是智商，Harness 是执行力，你才是决定方向的人。
