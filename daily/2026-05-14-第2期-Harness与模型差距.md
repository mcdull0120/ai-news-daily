## 🔥 今日大事 + 解读

### Anthropic 首次在 B2B 采用率上超越 OpenAI

**一句话摘要**：美国企业花钱使用 AI 的数据出炉了：用 Claude（Anthropic）的企业占 34.4%，首次超过用 ChatGPT（OpenAI）的 32.3%。而且 Claude 的企业客户数量在一年内翻了 4 倍。

**原文链接**：[The Decoder](https://the-decoder.com/anthropic-overtakes-openai-in-b2b-adoption-for-the-first-time-according-to-ramp-spending-data)（英文，需翻墙）

**💡 对你意味着什么：**

这条数据背后有三个信号：

1. **企业更看重"能干活的 AI"，不只是"能聊天的 AI"**  
   OpenAI 的 ChatGPT 名气最大、用户最多，但企业付费买的是"解决实际问题"。Claude 在编程、文档处理、复杂推理上的 Harness（驾驭能力）更强，所以企业愿意买单。

2. **B2B 市场看的是"性价比"和"可控性"**  
   Claude 的 API 价格策略更激进，而且对企业数据安全、合规性的支持更好。

3. **但领先优势可能不稳**  
   文章提到三个因素可能让 Claude 的优势减弱：OpenAI 正在追赶、Google Gemini 在 B2B 也在发力、企业采购决策有滞后性。

**这和今天的主题有什么关系？**  
这条数据证明了一件事：**"工具好不好用"（Harness）正在比"模型名不名气"（品牌）更重要。** Claude 能反超，不是因为 Claude 模型比 GPT-4 聪明很多，而是因为 Claude 的"身体"（Claude Code、API 稳定性、企业功能）更适合干活。

**⚠️ 知识截止说明**：以下概念解读基于我的训练知识（截止到 2025 年初）。Anthropic 反超 OpenAI 是 2026 年 5 月的新数据，我对具体数据的分析来自 AI HOT 提供的新闻摘要。

## 📚 今日概念

### 第一部分：Harness（驾驭能力）到底强在哪里

#### 第一层（浅）：从"聊天"到"干活"的进化

上一期我们讲了"大脑"和"身体"的区别。这期讲：**为什么"身体"的好坏，往往比"大脑"更能决定你的体验。**

想象两个助理：

**助理 A（普通对话 AI）：**
- 智商 140，哈佛毕业
- 你让他写个 PPT，他说："好的，这是 PPT 的结构大纲和内容。"
- 然后... 没了。你自己打开 PowerPoint，复制粘贴，调格式，找配图。

**助理 B（Claude Code）：**
- 智商 120，普通大学毕业
- 你让他写个 PPT，他说："好的。"
- 然后他自己打开 Gamma/Canva，生成 PPT，调好格式，插好图，发你链接。如果你说"字体太丑"，他自己改，改完再发你。

**你选谁？** 大多数人选 B。因为 B 的 Harness（驾驭系统）好——他不只是"会"，他是"能干"。

#### 第二层（中）：Harness 的 5 个核心组件

Harness 不是单一功能，而是一套"让大脑能干活"的系统。它包含 5 个核心组件：

| 组件 | 作用 | Claude Code 的例子 | 普通对话 AI 的局限 |
|------|------|-------------------|------------------|
| **1. 环境感知** | 读取周围的信息和文件 | 自动扫描你整个代码库，理解项目结构 | 只能你手动复制粘贴代码给他 |
| **2. 文件操作** | 读写、创建、删除文件 | 直接创建 index.html、style.css | 给你代码文本，你自己保存 |
| **3. 终端执行** | 运行命令、测试、部署 | 运行 `npm install`、启动服务器、跑测试 | 无法执行任何命令 |
| **4. 多步规划** | 把大任务拆成小步骤 | "做网站"→建文件→写代码→调样式→测试→修复 | 只能单轮回答，没有长期记忆和规划 |
| **5. 工具调用** | 调用外部 API、数据库、服务 | 通过 MCP/Skill 查新闻、操作数据库、发邮件 | 完全封闭，只能用自己的知识 |

**关键洞察**：
- 组件 1-3 是"基础体力"——让 AI 从"动嘴"变成"动手"
- 组件 4 是"脑力升级"——让 AI 从"回答问题"变成"完成任务"
- 组件 5 是"无限扩展"——让 AI 从"单机"变成"联网"

#### 第三层（深）：为什么 Claude Code 的 Harness 特别强

同样是"身体"，Claude Code 的身体为什么比别的工具强？三个原因：

**原因 1：自主执行循环（Agentic Loop）**

普通工具的流程：
```
你说 → 它答 → 结束
```

Claude Code 的流程：
```
你说 → 它理解 → 拆步骤 → 执行第 1 步 → 看结果 → 
→ 发现错误 → 自我修正 → 执行第 2 步 → ... → 完成 → 汇报
```

这就像：普通 AI 是"你问一句，我答一句"的客服；Claude Code 是"你给我目标，我自己搞定"的项目经理。

**原因 2：上下文窗口管理**

大模型都有"上下文窗口"（一次能看多少字的限制）。Claude Code 的 Harness 会自动判断：
- 这个任务需要看哪些文件？
- 文件太长，哪部分最重要？
- 多轮对话后，哪些信息不能丢？

普通工具需要你手动管理这些（复制粘贴、反复提醒），Claude Code 自己搞定。

**原因 3：工具生态（MCP + Skill）**

Claude Code 不靠自己学会所有技能，而是通过标准化接口（MCP）接入无限的外部工具：
- 需要查新闻？装 AI HOT Skill
- 需要操作数据库？装数据库 MCP
- 需要发 Slack 消息？装 Slack MCP

这就像 iPhone 的 App Store——手机本身功能有限，但装了 App 就无所不能。

### 第二部分：国内模型 vs 国外模型的真实差距

上一期你说"我用的模型还是国内的模型"。这期回答你：**国内模型和国外模型的差距到底在哪？值不值得"换发动机"？**

#### 第一层（浅）：直观感受的差距

| 维度 | 国外顶尖模型（GPT-4 / Claude 3.5） | 国内主流模型（文心一言 / 通义千问 / 讯飞星火） |
|------|----------------------------------|------------------------------------------|
| **回答质量** | 逻辑严密、举例贴切、能处理复杂推理 | 日常对话 OK，复杂问题容易"胡说"或绕圈子 |
| **创造力** | 写故事、想点子、做比喻很强 | 偏保守，创造性任务容易"标准答案化" |
| **代码能力** | 能写复杂项目、调试、优化 | 能写简单代码，复杂项目容易出错 |
| **长文本** | 能一次处理几十万字（Claude 200K） | 通常 4K-16K，长文档容易"遗忘"前文 |
| **诚实度** | 不知道就承认，不会硬编 | 有时会"幻觉"（自信地胡说） |

**但要注意**：这个对比是"顶尖国外模型" vs "国内平均水平"。国内也有很强的模型（比如 DeepSeek、Kimi、通义千问最新版），差距在缩小。

#### 第二层（中）：技术差距到底在哪

**差距 1：训练数据的质量和多样性**
- 国外模型：训练数据覆盖全球互联网，代码、论文、书籍、多语言内容极其丰富
- 国内模型：中文数据强，但英文技术文档、开源代码、学术论文的覆盖相对弱
- **结果**：写代码、读论文、处理技术问题时，国外模型明显更强

**差距 2：模型架构和对齐技术**
- 国外模型在"指令跟随"（你让它干嘛它干嘛）和"安全对齐"（既安全又不至于太保守）上投入更多
- 国内模型有时"对齐过度"——为了安全，拒绝回答很多正常问题，或者回答很模板化

**差距 3：推理能力（Chain-of-Thought）**
- 国外顶尖模型在复杂数学、逻辑推理、多步规划上更稳定
- 国内模型"一步推理"还行，"多步推理"容易在中间某步出错

#### 第三层（深）：实际使用中的体验差距

**场景 1：写代码**
```
你：帮我写一个能上传图片、识别文字、生成摘要的网页

GPT-4/Claude：生成完整代码，包含前端界面、后端接口、错误处理
国内模型：生成基础代码，缺少错误处理，接口可能不完整
```

**场景 2：读长文档**
```
你：这是一份 100 页的合同，帮我找出所有对甲方不利的条款

Claude（200K 上下文）：逐条分析，给出具体页码和修改建议
国内模型（16K 上下文）：只能读前 20 页，后面的"忘了"，或者 summary 太笼统
```

**场景 3：创意写作**
```
你：写一个科幻短篇，主角是一个发现自己其实是 AI 的人类

GPT-4/Claude：情节有转折、人物有深度、结尾有回味
国内模型：情节平铺直叙，容易变成"AI 科普文章"而不是小说
```

#### 给你的实用建议

**什么时候国内模型够用？**
- 日常聊天、查资料、写简单文案
- 处理中文内容（古文、诗词、国内政策解读）
- 对数据隐私要求极高（不想把数据发到国外）

**什么时候值得用国外模型？**
- 编程开发（代码能力差距明显）
- 复杂推理、数据分析
- 长文档处理
- 创意工作（写作、策划）
- 用 Claude Code / Cursor 等工具时（这些工具默认接国外模型，Harness 效果最佳）

**性价比方案（很多人这么做）：**
- 日常聊天 → 国内模型（快、便宜、中文好）
- 重要工作 → 国外模型（准、强、稳定）
- 编程开发 → 国外模型 + Claude Code/Cursor

## 🛠️ 今日技巧/案例：Harness 差距的直观对比

### 场景：同一个需求，不同工具的回应

**需求**："帮我分析一下这份销售数据，找出问题并给出建议"

**国内模型（网页版）：**
```
[给你一段通用的销售分析框架]
[建议：加强客户关系管理、优化产品线...]
❌ 没有读你的数据文件
❌ 建议是"套话"，不针对你的具体情况
```

**GPT-4（网页版）：**
```
[你手动复制粘贴数据后]
[分析比较具体，能指出几个数字异常]
⚠️ 但如果你数据量太大，复制粘贴很麻烦
⚠️ 如果你想让它画图，它只能给你代码，你自己运行
```

**Claude Code + Claude 模型：**
```
你：分析这份销售数据
Claude Code：
→ 读取 sales_2025.csv ... 完成（自动扫描文件）
→ 发现数据量 50 万行，自动采样分析 ... 完成
→ 发现 3 个异常：Q3 华南区下滑 40%、新品 A 退货率 25%、老客户复购率下降
→ 生成可视化图表 ... 完成
→ 给出针对性建议：华南区建议调研竞品 X、新品 A 建议暂停推广
→ 生成完整分析报告 report.html ... 完成
→ 打开浏览器展示报告
```

**差距总结**：不是"模型懂不懂分析"的问题，而是**"工具能不能读取文件、处理大数据、生成图表、输出报告"**的问题。这就是 Harness 的差距。
