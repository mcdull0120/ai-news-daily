# AI 小白进阶日报 · 写作风格指南

> 参考来源：卡兹克（khazix-writer）写作风格，适配为"AI 小白进阶日报"专用版。
> 核心原则：**保留小白友好的结构化设计，注入活人感和去 AI 味表达。**

---

## 一、去 AI 味禁令（必须遵守）

写完后通读一遍，遇到以下词汇/句式必须替换：

| 禁用词/句式 | 为什么禁用 | 替换方案 |
|---|---|---|
| "首先...其次...最后" | AI 味最浓的排比结构 | 用自然转场："说到这个""回到这块""顺着上面再聊聊" |
| "说白了" | AI 标志性口头禅 | "坦率的讲""其实就是""说到底" |
| "本质上" | 太学术，像论文 | "说到底""其实" |
| "换句话说" | 书面感过重 | "你想想看""也就是说" |
| "意味着什么？" / "这意味着" | AI 爱用的推理句式 | "那结果会怎样呢""所以呢" |
| "不可否认" | 套话 | 直接删掉，换成正面陈述 |
| "综上所述" / "总的来说" | 总结模板 | 用具体的回扣句替代 |
| "值得注意的是" / "不难发现" | 无意义的引导词 | 直接说，不要铺垫 |
| "让我们来看看..." / "接下来让我们..." | AI 典型的"导游句式" | 直接开始讲，不要召唤读者 |
| "在当今 AI 快速发展的时代" / "随着技术的不断进步" | 教科书式宏大开头 | 从一个具体事件/数据/场景切入 |
| "赋能、抓手、闭环、颗粒度" | 互联网黑话 | 用大白话替代 |
| "AI 工具" / "某个模型" / "相关技术" | 空泛指代 | 说具体名字：Claude、GPT-4、Kimi、Coze 等 |

### 额外注意
- **不要编造案例**："比如有一次..." 这种没有出处的编造是大忌。如果没真实案例，写"我自己还没试过，但想想就觉得..."
- **不要假设读者知道**：不要默认读者懂专业术语，遇到术语必须在前 20 字内给一个"人话版"解释。

## 二、口语化弹药库（主动使用）

这些词组自带"活人感"，不要每句都塞，在需要转场、表达观点、拉近距离时自然地用：

**转场过渡**
> 坦率的讲、说真的、怎么说呢、其实吧、你想想看、我跟你说、回到这块、顺着上面的再聊聊、这块需要注意一下

**表达判断**
> 我是真的觉得、反正我觉得、不是说这个不行而是说、我自己的感受是、我始终坚信、我觉得还是挺重要的

**承认和自嘲**
> 说实话我也不确定、我自己也还在摸索、可能有些想法还不成熟、这个事儿我也踩过坑、愚钝如我

**情绪表达**
> 这种感觉太爽了、我当时就愣住了、想想就觉得兴奋、我真的被震撼到了、太离谱了、给我一下子整不会了

**拉近距离**
> 很多朋友可能不知道、可能有小伙伴纳闷、你如果关注这个领域的话、大家也都知道

**注意**：日报的情绪浓度要比卡兹克原文低一档。他是个人公众号，可以"太特么刺激了"；你是学习产品，情绪表达要克制在"我当时就愣住了"这种程度，避免过于口语化导致不专业。

## 三、五段式写作要点

### 01 🔥 近期 Hot 事件

**目标**：让读者在 30 秒内知道"发生了什么"和"为什么跟我有关"。

**写作要点**：
- **开头直接丢事实**：不要铺垫，第一句就是核心数据或结论。"美国企业 AI 支出数据出炉了：Claude 的企业占比首次超过 OpenAI。"
- **数据要口语化翻译**：不要只写"34.4% vs 32.3%"，要解释"这意味着什么"——"别小看这 2 个百分点的差距，这意味着企业采购的风向标变了。"
- **延伸启发必须落回读者**：企业数据→个人用户能学到什么？用"对你我来说"做转折。

**卡兹克风格适配**：
- 用"坦率的讲"引出核心判断
- 用"当时我就..."的体感句式代替"值得注意的是"

---

### 02 📚 概念学习

**目标**：把复杂概念用"人话"讲清楚，让小白读完能跟朋友复述。

**写作要点**：
- **永远从问题出发**：不要直接下定义，先问"为什么同样的模型，在不同工具里体验差这么多？"——这是概念学习的钩子。
- **类比优先于定义**：用生活化类比（助理 A vs 助理 B、木匠比赛、健身教练）代替抽象解释。
- **知识要"顺手掏出来"**：不要写"下面我来介绍 Harness 的五个核心组件"，要写"Harness 不是单一功能，它其实是一套系统，我把它拆成 5 个部分，你会发现生活里处处都是这套逻辑。"
- **每段结尾要回扣**：讲完一个组件，用一句话拉回主线"你看，这就是 Harness 的第一块拼图。"

**卡兹克风格适配**：
- 用"聊着聊着顺手掏出来"的方式输出知识
- 用具体的"你让他写个 PPT"场景代替抽象描述

---

### 03 🛠️ 技巧与案例

**目标**：用同一个需求丢给不同工具的"回应模拟"，让读者秒懂差距。

**写作要点**：
- **对比要极端化**：三种工具的回应差距要拉得足够大，不能都差不多。差的要写得让人摇头，好的要写得让人"卧槽"。
- **逐一展示法（升番逻辑）**：先展示最差的（国内模型），再展示中等的（GPT-4），最后展示最好的（Claude Code）。情绪要一轮一轮往上推。
- **每个案例配一句人话点评**：不要只罗列功能差异，要用"这就像..."的比喻让人秒懂。

**卡兹克风格适配**：
- 用"逐一展示法"带吐槽点评，制造发现感
- 用"不是...而是..."的反向论证制造反差

---

### 04 💡 灵魂洞察

**目标**：不只讲技术，讲人与工具的关系。这是整期最"卡兹克"的部分。

**写作要点**：
- **必须有一个类比/故事**：木匠做椅子、杠杆原理、代笔者 vs 杠杆——洞察不能是抽象说教，必须包裹在故事里。
- **先理解读者再给出视角**：先站读者角度承认焦虑（"很多人担心用 AI 多了会变笨"），再说"但关键不在用不用，而在你怎么用"。
- **只保留一处升华**：不要开头总结一次、中间再总结一次、结尾再升华一次。引言和结尾只留一个。
- **情绪要真实**：用"我当时就愣住了"代替"我感到非常震撼"，用体感记忆代替知识性描述。

**卡兹克风格适配**：
- 用"人物画像法"从一个数据点想象背后具体的人（但日报里可以简化为"想象一下，你是一个..."）
- 用"回环呼应"在结尾 callback 开头的意象
- **这是全文最可以"卡兹克化"的一段**，口语化浓度可以最高

---

### 05 🗣️ 今日一问

**目标**：收集反馈，决定下期内容方向。

**写作要点**：
- **只问一个核心问题**：不要罗列 4 个问题，只问"你还有哪些想了解的？"
- **选项要有温度**：不要冷冰冰的 A/B/C/D，要写"暂无，等下期看看 👀""想看 Claude Code 的具体实操教程"
- **必须给反馈闭环**：提交按钮 + "反馈已收到"提示，让用户知道他的声音被听到了。
- **支持多选**：允许用户选多个方向，但提交后统一归入"内容生成库"，下期只取一个方向深挖。

## 七、内容选题与反馈处理流程（测试期版）

### 反馈优先级（从高到低）

| 来源 | 优先级 | 处理方式 |
|---|---|---|
| 用户输入的**自定义想法** | 最高 | 直接作为下期核心方向 |
| 多选票数**最多的选项** | 高 | 作为下期主题，其余选项进"内容生成库" |
| AI HOT 热点新闻 | 中 | 作为"事件"板块的素材来源 |
| 编辑自主判断 | 低（测试期） | 仅在用户无明确反馈时使用 |

### 反馈处理规则

1. **无效输入过滤**：用户输入无意义或无法读取的内容 → 直接忽略，不影响选题。
2. **多选不丢弃**：用户选了多个方向 → 全部纳入"内容生成库"（按时间排序），下期只取其中一个执行，剩下的按库内顺序排期。
3. **单期聚焦**：每期只围绕**一个核心话题**展开，不要试图在一期里覆盖所有用户兴趣。
4. **知识递进**：相邻两期之间要有连续性。例如第 2 期讲 Harness 概念 → 第 3 期讲 Claude Code 实操（概念的落地）。

### 内容生成工作流（测试期）

```
Step 1：fetch_data.py 抓取 AI HOT 数据 → 初筛 3-5 条候选新闻
   ↓
Step 2：结合用户反馈 + 内容生成库 → 确定下期核心方向
   ↓
Step 3：AI 按"写作风格指南"生成初稿（5 段式结构 + 卡兹克风格）
   ↓
Step 4：人工二审（重点审：核心角度、个人经历/情绪、AI 味扫描）
   ↓
Step 5：套 issue-v3 模板 → 输出 HTML → 本地预览 → 发布
```

**人工二审重点**：
- 核心创意角度是否是"只有你能写出来的视角"？
- 有没有第一手观察或真实经历？（AI 编造的要删掉或改写）
- 情绪表达是"体感记忆"还是"知识性描述"？
- L1 禁用词是否零命中？

## 四、格式保留声明（这些不能动）

以下设计和卡兹克风格冲突，但对你的产品定位更重要，**必须保留**：

- ✅ **保留 5 段式标题框**：小白需要导航，不能隐藏结构
- ✅ **保留 emoji 标题**：🔥📚🛠️💡🗣️ 是产品的视觉标识
- ✅ **保留结构化卡片**：概念组件列表、技巧对比卡片、选项按钮——这些是学习产品的交互设计
- ✅ **保留冒号、双引号**：模板视觉设计已大量使用，改掉会破坏一致性
- ✅ **保留阅读时间标注**：学习产品需要管理用户预期

## 五、出稿前自检清单（3 分钟版）

写完一期内容后，按以下顺序快速过一遍：

### L1 硬性规则（1 分钟）
- [ ] 全文搜索"说白了""本质上""首先""值得注意的是"——出现则替换
- [ ] 检查有没有"在当今 AI 快速发展的时代"这类宏大开头——有则重写第一句
- [ ] 检查概念部分有没有"下面我来介绍"——改为"这让我想起..."

### L2 活人感扫描（1 分钟）
- [ ] 读一遍，有没有连续 3 句以上句式长度一样？有则打破节奏
- [ ] 有没有至少 2 处口语化表达（"坦率的讲""说真的""我当时就..."）？
- [ ] 洞察部分有没有先理解读者再给出视角？（"很多人担心...""但关键在..."）

### L3 结构检查（1 分钟）
- [ ] 事件部分：延伸启发是否落回了个人用户？
- [ ] 概念部分：有没有用类比？有没有术语的人话解释？
- [ ] 技巧部分：三种工具的差距是否足够极端？
- [ ] 洞察部分：引言和结尾是否只保留了一处升华？
- [ ] 全文字数是否控制在 15 分钟阅读以内？

**通过标准**：L1 零命中，L2 至少通过 2/3，L3 全部通过。

## 八、已确认的产品决策（需持续遵守）

- **测试期坚持人工二审**：AI 生成初稿后必须由人审校，稳定运行前不自动发布。
- **反馈处理规则**：无效输入直接忽略；多选方向全部进"内容生成库"按序排期；单期只聚焦一个核心话题。
- **知识递进原则**：相邻两期之间有连续性，先讲概念再讲实操，不要跳来跳去。

## 六、示例对比

### ❌ AI 味原文
> 首先，我们需要了解 Harness 的概念。Harness 是 AI 工具的驾驭能力，本质上是指模型与外部系统的交互能力。值得注意的是，Claude Code 的 Harness 特别强，这意味着它在企业场景中具有显著优势。

### ✅ 适配版
> 说实话，我第一次听到 Harness 这个词也愣了一下。啥意思？说白了就是：你的 AI 大脑再聪明，如果身体动不了，那也白搭。Claude Code 之所以能让企业买单，不是因为它比 GPT-4 聪明多少，而是因为它长了手——能读文件、能跑命令、能自己改错。
