# AI 小白进阶日报 — 项目手册

> 本文件为项目级记忆文档，记录产品定位、设计规范、内容标准和技术实现。任何结构性变更须经确认后更新此处。

---

## 1. 产品定位

| 维度 | 定义 |
|------|------|
| **名称** | AI 小白进阶日报 |
| **受众** | AI 初学者 / 非技术背景小白 |
| **频率** | 日报（测试期） |
| **核心定位** | "新闻事件 + 概念讲解 + 案例对比 + 人文洞察" 的进阶学习读物 |
| **差异化** | 不是纯资讯罗列，而是帮小白理解"这条新闻跟我有什么关系" |
| **载体** | 手机优先的网页版（H5），`max-width: 540px` |
| **当前阶段** | 测试期：人工二审 + 收集反馈 + 迭代模板 |

### 1.1 要解决的核心问题
1. **信息过载**：AI 新闻太多，小白不知道哪些值得看
2. **看不懂**：专业术语密集，没有背景知识读不下去
3. **不会用**：知道发生了什么，但不知道怎么应用
4. **没体感**：纯文字堆砌，读完留不下印象

### 1.2 产品承诺
- 每天精选 **3 条** 最值得看的新闻
- 每条新闻回答三个问题：**发生了什么？为什么值得关注？对你有什么启发？**
- 每期围绕 **1 个核心概念** 展开，从新闻自然引出
- 用 **同一个需求 + 不同工具对比** 的方式，让你秒懂差距
- 最后留一个 **人文洞察**，不只讲技术，讲人与工具的关系

## 2. 版本定义（核心拆分）

### 2.1 轻度版
- **文件**：`public/daily-brief.html`
- **内容**：AI 新闻 + 简单分析
- **定位**：快速浏览，轻量化阅读
- **视觉**：紫色渐变头图 + 卡片式新闻列表
- **标签体系**：重磅（绿）/ 突破（蓝灰）/ 趣味（驼）

### 2.2 深度版
- **文件**：`public/deep-dive-news.html`
- **内容**：AI 新闻 → 知识深潜 → 工具使用 → 洞察 → 提问用户
- **定位**：完整进阶学习路径，对应原 4 段式结构。新闻与轻度版**不重复**，可以是轻度版某条的深层关联，也可以是用户兴趣方向的独立选题
- **选题原则**：适合该用户画像的、重要的、最新的、用户感兴趣方向的
- **新闻格式**：新闻简述 → 背景补充 → 为什么值得关注 → 阅读原文链接

## 3. 内容结构（深度版）

深度版沿用 v1.0 确立的 **4 段式结构**：

| 序号 | 区块 | emoji | 功能 | 内容要求 |
|------|------|-------|------|----------|
| 01 | AI 大事件 | 🔥 | 新闻入口 | 3 条滑动卡片（重磅/突破/趣味），带底部指示点 |
| 02 | 概念 & 案例 | 📚 | 知识核心 | 概念讲解 + 真实案例对比（❌⚠️✅ 三档） |
| 03 | 灵魂洞察 | 💡 | 人文升华 | 故事/类比 + 人与工具的关系 + 只留一处升华 |
| 04 | 今日一问 | 🗣️ | 反馈收集 | 单问题 + 4 选项 + 自定义输入 + 提交按钮 |

### 3.1 各区块写作要点

**AI 大事件**
- 开头直接丢事实，第一句就是核心数据或结论
- 数据要口语化翻译（不要只写"34.4% vs 32.3%"，要解释"这意味着什么"）
- 延伸启发必须落回读者："对你我来说"
- 呈现形式：横向滑动卡片（CSS scroll-snap，纯 CSS 无 JS）

**概念 & 案例**
- 永远从问题出发，不要直接下定义
- 类比优先于定义（客服 vs 上门师傅、电钻 vs 手摇钻）
- 知识要"顺手掏出来"，不要"下面我来介绍"
- 案例对比要极端化，三档差距拉得足够大（差的摇头，好的"卧槽"）
- 升番逻辑：先最差 → 再中等 → 最后最好，情绪逐轮推高

**灵魂洞察**
- 必须有一个类比/故事包裹观点（木匠、电钻、杠杆、椅子）
- 先站读者角度承认焦虑（"很多人担心..."），再说"但关键在..."
- 只保留一处升华（引言和结尾只留一个）
- 情绪真实，用体感记忆代替知识性描述

**今日一问**
- 只问 1 个核心问题
- 选项要有温度，带 emoji 或口语化表达
- 支持多选（checkbox），全部进"内容生成库"
- 提交按钮点击后变为 `✅ 反馈已收到，谢谢！` 并置灰

## 4. 视觉规范

### 4.1 莫兰迪配色

| 用途 | 色值 |
|------|------|
| 全局背景 | `#F0EBE4`（暖灰白） |
| 事件区块 | `#FFFFFF`（纯白） |
| 概念区块 | `#F5F0E8`（暖米） |
| 技巧区块 | `#E8F0E0`（浅灰绿） |
| 洞察区块 | `linear-gradient(180deg, #E0D5C8, #D5C8B8)`（暖棕渐变） |
| 提问区块 | `#FFFFFF` |
| 文字主色 | `#3D3530`（深棕） |
| 文字次色 | `#7A7168` / `#9A9188` |
| 主色（莫兰迪绿） | `#A8B5A0` |
| 次要（莫兰迪驼） | `#B8A99A` |
| 强调（莫兰迪蓝灰） | `#9AAFB7` |

### 4.2 排版铁律
1. **居中大标题**：若采用居中对齐，必须确保不会出现"一行只有一个字/标点"的情况。解决方案：缩小字体、缩短文字、用 `<br>` 精确控制换行位置
2. **洞察引言**：必须左对齐（`text-align: left`），居中会导致换行丑陋
3. **任何居中对齐的短文本**：需预判手机 375px 宽度下的换行结果
4. **同一块区域内不要混用多个字体大小**，除非有明确的层级必要性
5. **标题换行参考**：24px 中文字体在 375px 宽度下约每行 15 个字；在 540px 下约每行 20 个字

### 4.3 标题框规范（section-header）
- 白色圆角卡片（`border-radius: 14px`）
- `box-shadow: 0 2px 12px rgba(61,53,48,0.08)`
- 左侧 4px 彩色竖线：事件 `#A8B5A0`（绿）、概念 `#B8A99A`（驼）、洞察 `#C4A882`（棕）
- 左侧圆形编号（36px，渐变背景）
- 标题 20px 加粗，副标题 13px 灰色

## 5. 写作风格（卡兹克适配版）

### 5.1 去 AI 味禁令（L1 硬性规则）

写完后通读一遍，遇到以下词汇/句式必须替换：

| 禁用词/句式 | 替换方案 |
|-------------|----------|
| "首先...其次...最后" | 用自然转场："说到这个""回到这块""顺着上面再聊聊" |
| "说白了" | "坦率的讲""其实就是""说到底" |
| "本质上" | "说到底""其实" |
| "换句话说" | "你想想看""也就是说" |
| "意味着什么？" / "这意味着" | "那结果会怎样呢""所以呢" |
| "不可否认" | 直接删掉，换成正面陈述 |
| "综上所述" / "总的来说" | 用具体的回扣句替代 |
| "值得注意的是" / "不难发现" | 直接说，不要铺垫 |
| "让我们来看看..." / "接下来让我们..." | 直接开始讲，不要召唤读者 |
| "在当今 AI 快速发展的时代" | 从一个具体事件/数据/场景切入 |
| "赋能、抓手、闭环、颗粒度" | 用大白话替代 |
| "AI 工具" / "某个模型" | 说具体名字：Claude、GPT-4、Kimi、Coze 等 |

### 5.2 口语化弹药库（主动使用）

**转场过渡**：坦率的讲、说真的、怎么说呢、其实吧、你想想看、我跟你说、回到这块、顺着上面的再聊聊

**表达判断**：我是真的觉得、反正我觉得、不是说这个不行而是说、我自己的感受是

**承认和自嘲**：说实话我也不确定、我自己也还在摸索、这个事儿我也踩过坑

**情绪表达**：我当时就愣住了、想想就觉得兴奋、我真的被震撼到了、太离谱了

**注意**：日报的情绪浓度要比卡兹克原文低一档，核心内容保持严肃。

## 6. 信息来源

### 6.1 数据源
- **主数据源**：AI HOT（aihot.virxact.com）公共 API
- **脚本**：`fetch_data.py`
  - 拉取最近 7 天精选内容（`/api/public/items?mode=selected`）
  - 过滤敏感词（死亡、犯罪、诉讼、裁员等 18 类）
  - 过滤过深技术内容（benchmark、梯度、蒸馏等，techScore ≥ 3 排除）
  - 按兴趣标签分类：创意、应用、文案、教育、提效
  - 每类最多取 2 条，最终精选 6-8 条
  - 输出 `public/data.json`

### 6.2 内容筛选标准
1. **时效性**：优先当天/昨天发布的新闻
2. **相关性**：至少有一条能引出本期核心概念
3. **多样性**：3 条卡片尽量覆盖重磅/突破/趣味三种类型
4. **来源可信度**：优先官方博客、权威媒体
5. **禁选**：纯技术论文（除非能用人话讲清楚）、负面/争议事件（测试期避免）、过于垂直的行业动态

## 7. 内容生产工作流

```
Step 1: python3 fetch_data.py → 抓取 AI HOT 数据 → 初筛候选新闻
   ↓
Step 2: 结合用户反馈 + 内容生成库 → 确定下期核心方向
   ↓
Step 3: AI 按写作指南生成初稿（4段式结构 + 卡兹克风格）
   ↓
Step 4: 人工二审（核心角度、真实经历、AI味扫描、L1禁用词）
   ↓
Step 5: 套 issue-v3 模板 → 输出 HTML → 本地预览 → 发布
```

**人工二审重点**：
- 核心创意角度是否是"只有你能写出来的视角"？
- 有没有第一手观察或真实经历？（AI 编造的要删掉或改写）
- 情绪表达是"体感记忆"还是"知识性描述"？
- L1 禁用词是否零命中？

### 7.1 反馈处理规则
- **无效输入过滤**：用户输入无意义内容 → 直接忽略，不影响选题
- **多选不丢弃**：用户选的多个方向 → 全部纳入"内容生成库"（按时间排序），下期只取其中一个执行，剩下的按库内顺序排期
- **单期聚焦**：每期只围绕 **1 个核心话题** 展开
- **知识递进**：相邻两期之间要有连续性。先讲概念再讲实操，不要跳来跳去

## 9. 已知限制（待改进）

1. **轮播无 JS 交互**：底部圆点指示器不会随滑动自动更新
2. **Quiz 数据无持久化**：点击提交只是前端变字，没有后端接收
3. **每期 HTML 独立**：没有统一的模板渲染系统，每期都是手工复制修改
4. **图片缺失**：目前纯文字排版，缺少新闻配图、概念图解
5. **分享功能缺失**：没有微信分享卡片、生成长图等功能
6. **无导航/目录**：长文滚动时看不到当前读到哪个区块
7. **内容生成慢**：AI 生成初稿耗时 7-8 分钟，token 消耗高，需优化 Prompt 和生成策略
