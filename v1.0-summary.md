# AI 小白进阶日报 · v1.0 完整总结

> 生成日期：2026-05-15
> 适用版本：issue-v3 模板及之前全部迭代成果
> 用途：2.0 迭代前的基线存档，方便对比与复盘

## 一、产品定位与目标

| 维度 | 定义 |
|---|---|
| **名称** | AI 小白进阶日报 |
| **受众** | AI 初学者 / 小白用户（非技术背景为主） |
| **频率** | 日报（测试期），每期预计阅读 10-15 分钟 |
| **核心定位** | "新闻事件 + 概念讲解 + 案例对比 + 人文洞察" 的进阶学习读物 |
| **差异化** | 不是纯资讯罗列，而是帮小白理解"这条新闻跟我有什么关系" |
| **载体** | 手机优先的网页版（H5），计划未来可嵌入小程序/公众号 |
| **当前阶段** | 测试期：人工二审 + 收集反馈 + 迭代模板 |

### 1.1 要解决的核心问题
1. **信息过载**：AI 新闻太多，小白不知道哪些值得看
2. **看不懂**：专业术语密集，没有背景知识读不下去
3. **不会用**：知道发生了什么，但不知道怎么应用到自己的工作和学习
4. **没体感**：纯文字堆砌，读完留不下印象

### 1.2 产品承诺
- 每天只精选 **3 条** 最值得看的新闻
- 每条新闻都回答三个问题：**发生了什么？为什么值得关注？对你有什么启发？**
- 每期围绕 **1 个核心概念** 展开，从新闻自然引出
- 用 **同一个需求 + 不同工具对比** 的方式，让你秒懂差距
- 最后留一个 **人文洞察**，不只讲技术，讲人与工具的关系

## 二、框架结构（4 段式）

v1.0 最终确定的是 **4 段式结构**，由最初的 5 段式合并而来（将"概念学习"和"技巧案例"合并为一个板块）。

| 序号 | 区块 | emoji | 功能 | 内容要求 |
|---|---|---|---|---|
| 01 | AI 大事件 | 🔥 | 新闻入口 | 3 条滑动卡片（重磅/突破/趣味），带底部指示点 |
| 02 | 概念 & 案例 | 📚 | 知识核心 | 概念讲解 + 真实案例对比（❌⚠️✅ 三档） |
| 03 | 灵魂洞察 | 💡 | 人文升华 | 故事/类比 + 人与工具的关系 + 只留一处升华 |
| 04 | 今日一问 | 🗣️ | 反馈收集 | 单问题 + 4 选项 + 自定义输入 + 提交按钮 |

### 2.1 各区块详细规范

#### 01 AI 大事件
- **呈现形式**：横向滑动卡片（CSS scroll-snap，纯 CSS 无 JS）
- **卡片数量**：固定 3 张
- **标签体系**：
  - 重磅（莫兰迪绿 #E8F0E0）—— 行业格局变化、重要数据发布
  - 突破（莫兰迪蓝灰 #E8ECF0）—— 技术/产品新进展
  - 趣味（莫兰迪驼 #F5F0E8）—— 社会现象、反常识观察
- **每张卡片包含**：标签 → 标题 → 时间/来源 → 摘要 → 原文链接 → "为什么值得关注"分析（2 条 bullet）
- **区块底部**：`event-summary` 总结框，提炼 3 条新闻的共同点，回扣本期主题

#### 02 概念 & 案例
- **引入框**：`concept-intro-box`，承接上期内容，提出本期要解决的问题
- **概念部分**：
  - 永远从问题出发，不要直接下定义
  - 类比优先于定义（客服 vs 上门师傅、电钻 vs 手摇钻）
  - 知识要"顺手掏出来"，不要"下面我来介绍"
  - 组件/清单用编号圆角卡片展示
  - 每段结尾回扣主线
- **过渡框**：`transition-box`，连接概念和案例，降低阅读跳跃感
- **案例部分**：
  - 同一个需求丢给三种工具（国内模型网页版 / GPT-4 网页版 / Claude Code）
  - 三档差距必须拉得足够大（差的摇头，好的"卧槽"）
  - 升番逻辑：先最差 → 再中等 → 最后最好，情绪逐轮推高
  - 每个案例配一句人话点评
  - 底部 `demo-conclusion` 总结框
- **小节框**：`concept-outro-box`，一句话总结本节

#### 03 灵魂洞察
- **引言框**：`insight-quote-main`，左对齐（**禁止居中**，防止换行丑陋），3-4 句诗化短句，最后一句加粗
- **正文**：
  - 必须有一个类比/故事（木匠、电钻、杠杆、椅子）
  - 先站读者角度承认焦虑（"很多人担心..."），再说"但关键在..."
  - 只保留一处升华（引言和结尾只留一个）
  - 情绪真实，用体感记忆代替知识性描述
  - 这是全文口语化浓度最高的一段

#### 04 今日一问
- **问题数量**：只问 1 个核心问题
- **选项设计**：4 个，要有温度（不要冷冰冰的 A/B/C/D）
- **支持多选**：checkbox，非 radio
- **自定义输入**：placeholder 提示
- **提交按钮**：点击后变为 `✅ 反馈已收到，谢谢！` 并置灰
- **提示文案**：按钮下方加一行小字说明反馈用途

## 四、信息获取来源

### 4.1 数据源
- **主数据源**：AI HOT（aihot.virxact.com）公共 API
- **脚本**：`fetch_data.py`
  - 拉取最近 7 天精选内容（`/api/public/items?mode=selected`）
  - 过滤敏感词（死亡、犯罪、诉讼、裁员等 18 类）
  - 过滤过深技术内容（benchmark、梯度、蒸馏等，techScore ≥ 3 排除）
  - 按兴趣标签分类：创意、应用、文案、教育、提效
  - 每类最多取 2 条，最终精选 6-8 条
  - 输出 `public/data.json`

### 4.2 内容筛选标准
1. **时效性**：优先当天/昨天发布的新闻
2. **相关性**：至少有一条能引出本期核心概念
3. **多样性**：3 条卡片尽量覆盖重磅/突破/趣味三种类型
4. **来源可信度**：优先官方博客、权威媒体（The Decoder、Anthropic Newsroom、OpenAI 官网）
5. **禁选**：纯技术论文（除非能用人话讲清楚）、负面/争议事件（测试期避免）、过于垂直的行业动态

### 4.3 人工二审要点（测试期强制）
- 核心创意角度是否是"只有你能写出来的视角"？
- 有没有第一手观察或真实经历？（AI 编造的要删掉或改写）
- 情绪表达是"体感记忆"还是"知识性描述"？
- L1 禁用词是否零命中？

## 五、UI 设计与排版规范

### 5.1 整体布局
- **容器**：`max-width: 540px`，居中，`margin: 0 auto`
- **手机优先**：所有设计以 375px-430px 宽度为基准
- **边距**：区块内边距 `36px 24px`，小屏幕降至 `20px`
- **字体**：系统默认无衬线字体栈（-apple-system, PingFang SC, Microsoft YaHei 等）
- **字号基准**：`html { font-size: 16px }`，小屏幕 `@media (max-width: 400px)` 降至 15px

### 5.2 莫兰迪配色方案

| 用途 | 色值 | 说明 |
|---|---|---|
| 全局背景 | `#F0EBE4` | 暖灰白 |
| 事件区块背景 | `#FFFFFF` | 纯白 |
| 概念区块背景 | `#F5F0E8` | 暖米 |
| 技巧区块背景 | `#E8F0E0` | 浅灰绿 |
| 洞察区块背景 | `linear-gradient(180deg, #E0D5C8, #D5C8B8)` | 暖棕渐变 |
| 提问区块背景 | `#FFFFFF` | 纯白 |
| 文字主色 | `#3D3530` | 深棕 |
| 文字次色 | `#7A7168` / `#9A9188` | 中灰棕 |
| 主色（莫兰迪绿） | `#A8B5A0` | 标题框、编号、按钮 |
| 次要（莫兰迪驼） | `#B8A99A` | 概念区块边框、分割线 |
| 强调（莫兰迪蓝灰） | `#9AAFB7` | 链接、渐变辅助 |
| 新闻标签-重磅 | `#E8F0E0` 底 + `#5A6B52` 字 | 绿色系 |
| 新闻标签-突破 | `#E8ECF0` 底 + `#5A6B6E` 字 | 蓝灰系 |
| 新闻标签-趣味 | `#F5F0E8` 底 + `#8A7568` 字 | 驼色系 |

### 5.3 标题框规范（section-header）
- 白色圆角卡片（`border-radius: 14px`）
- `box-shadow: 0 2px 12px rgba(61,53,48,0.08)`
- 左侧 4px 彩色竖线，每区块颜色不同：
  - 事件：`#A8B5A0`（绿）
  - 概念：`#B8A99A`（驼）
  - 洞察：`#C4A882`（棕）
- 左侧圆形编号（36px，渐变背景）
- 标题 20px 加粗，副标题 13px 灰色

### 5.4 排版铁律（防 orphan）
1. **居中大标题**：若采用居中对齐，必须确保不会出现"一行只有一个字/标点"的情况。解决方案：缩小字体、缩短文字、用 `<br>` 精确控制换行位置
2. **洞察引言**：必须左对齐（`text-align: left`），居中会导致换行丑陋
3. **任何居中对齐的短文本**：需预判手机 375px 宽度下的换行结果，必要时缩短文字或改左对齐
4. **同一块区域内不要混用多个字体大小**，除非有明确的层级必要性
5. **英文单词/品牌名**：在中文标题中注意其视觉宽度（约为中文字符 1.5 倍），换行计算时需换算
6. **标题换行参考**：24px 中文字体在 375px 宽度下约每行 15 个字；在 540px 下约每行 20 个字

### 5.5 新闻轮播交互
- 纯 CSS 实现，无 JavaScript
- `display: flex` + `overflow-x: auto` + `scroll-snap-type: x mandatory`
- 卡片宽度：`flex: 0 0 85%`，留出下一张的边缘暗示
- 底部圆点指示器：3 个圆点，active 状态变长（width 20px，圆角 4px）
- 隐藏滚动条：`::-webkit-scrollbar { display: none }`

### 5.6 Quiz 交互
- 选项卡片：白色圆角，点击时 `active` 背景变深
- Checkbox 隐藏，用自定义圆形 marker 替代
- 选中时 marker 变绿 + 显示白色对勾
- 提交按钮：绿色渐变 + 阴影 + 顶部高光伪元素，营造微 3D 感
- 点击后按钮置灰，文字变为"✅ 反馈已收到，谢谢！"

## 六、写作风格规范（卡兹克适配版）

### 6.1 去 AI 味禁令（L1 硬性规则）

| 禁用词/句式 | 替换方案 |
|---|---|
| "首先...其次...最后" | 用自然转场："说到这个""回到这块""顺着上面再聊聊" |
| "说白了" | "坦率的讲""其实就是""说到底" |
| "本质上" | "说到底""其实" |
| "换句话说" | "你想想看""也就是说" |
| "意味着什么？" / "这意味着" | "那结果会怎样呢""所以呢" |
| "不可否认" | 直接删掉，换成正面陈述 |
| "综上所述" / "总的来说" | 用具体的回扣句替代 |
| "值得注意的是" / "不难发现" | 直接说，不要铺垫 |
| "让我们来看看..." / "接下来让我们..." | 直接开始讲，不要召唤读者 |
| "在当今 AI 快速发展的时代" | 从一个具体事件/数据/场景切入 |
| "赋能、抓手、闭环、颗粒度" | 用大白话替代 |
| "AI 工具" / "某个模型" | 说具体名字：Claude、GPT-4、Kimi、Coze 等 |

### 6.2 口语化弹药库（主动使用）
- **转场**：坦率的讲、说真的、怎么说呢、其实吧、你想想看、我跟你说、回到这块、顺着上面再聊聊
- **表达判断**：我是真的觉得、反正我觉得、不是说这个不行而是说、我自己的感受是
- **承认和自嘲**：说实话我也不确定、我自己也还在摸索、这个事儿我也踩过坑
- **情绪表达**：我当时就愣住了、想想就觉得兴奋、我真的被震撼到了、太离谱了
- **拉近距离**：很多朋友可能不知道、可能有小伙伴纳闷、你如果关注这个领域的话

**注意**：日报的情绪浓度要比卡兹克原文低一档，核心内容保持严肃。

### 6.3 各区块写作要点

#### 事件板块
- 开头直接丢事实，第一句就是核心数据或结论
- 数据要口语化翻译（不要只写"34.4% vs 32.3%"，要解释"这意味着什么"）
- 延伸启发必须落回读者："对你我来说"

#### 概念板块
- 永远从问题出发，先问"为什么"再讲"是什么"
- 类比优先于定义
- 每段结尾回扣主线
- 术语必须在出现的前 20 字内给"人话版"解释

#### 案例板块
- 对比要极端化，三档差距拉得足够大
- 逐一展示法（升番逻辑），情绪逐轮推高
- 每个案例配一句人话点评，用"这就像..."的比喻

#### 洞察板块
- 必须有一个类比/故事包裹观点
- 先理解读者焦虑，再给出视角
- 只保留一处升华
- 情绪真实，用体感记忆代替知识性描述

## 八、反馈机制

### 8.1 Quiz 选项设计原则
- 必须有"暂无，等下期看看"选项，降低参与门槛
- 必须有"其他"自定义输入，捕捉未预见的用户需求
- 选项文案要有温度，带 emoji 或口语化表达
- 支持多选（全部进"内容生成库"）

### 8.2 反馈处理优先级
| 来源 | 优先级 | 处理方式 |
|---|---|---|
| 用户输入的自定义想法 | 最高 | 直接作为下期核心方向 |
| 多选票数最多的选项 | 高 | 作为下期主题，其余进内容生成库 |
| AI HOT 热点新闻 | 中 | 作为"事件"板块素材 |
| 编辑自主判断 | 低（测试期） | 仅在用户无明确反馈时使用 |

### 8.3 内容生成库规则
- 多选不丢弃：用户选的多个方向全部入库，按时间排序
- 单期聚焦：每期只围绕 1 个核心话题
- 知识递进：相邻两期有连续性（概念 → 实操 → 进阶）

## 十、v1.0 已知限制（2.0 可改进）

1. **轮播无 JS 交互**：底部圆点指示器不会随滑动自动更新（纯 CSS 无法监听 scroll 事件）
2. **Quiz 数据无持久化**：点击提交只是前端变字，没有后端接收和存储
3. **每期 HTML 独立**：没有统一的模板渲染系统，每期都是手工复制修改
4. **图片缺失**：目前纯文字排版，缺少新闻配图、概念图解等视觉元素
5. **分享功能缺失**：没有微信分享卡片、生成长图等功能
6. **无导航/目录**：长文滚动时看不到当前读到哪个区块
7. **本地预览依赖局域网 IP**：出门换 WiFi 后手机无法访问，需要内网穿透方案
8. **无阅读进度提示**：用户不知道读到哪里了
9. **无夜间模式**：只有一套莫兰迪配色
10. **内容生成依赖人工**：从选题到成稿需要大量人工介入，无法自动化
